常用注解

@RestController 和 @Controller

  • 相同点
  1. 元注解：@Target({ElementType.TYPE})，作用在类、接口、枚举类上
  2. 元注解：@Retention(RetentionPolicy.RUNTIME)，运行期保留
  3. 元注解：@Documented，保留文档注解信息
  • 不同点
    • Controller： 注册为一个控制器类
    • RestController: 等于 @Controller + @ResponseBody 注解

@ResponseBody

元注解：

  1. @Target({ElementType.TYPE, ElementType.METHOD})，作用在类和方法上
  2. @Retention(RetentionPolicy.RUNTIME)，运行期保留
  3. @Documented，保留文档注解信息

作用：

将controller的方法返回的对象通过适当的转换器转换为指定的格式之后，写入到response对象的body区，通常用来返回JSON数据或者是XML数据。

@RequestMapping

元注解：

  1. @Target({ElementType.TYPE, ElementType.METHOD})，作用在类和方法上
  2. @Retention(RetentionPolicy.RUNTIME)，运行期保留
  3. @Documented，保留文档注解信息

作用：

在Spring MVC 中使用 @RequestMapping 来映射请求，相当于配置了映射请求

@GetMapping

RequestMapping的method=”get”，作用只能在方法上

@PostMapping

RequestMapping的method=“post”，作用只能在方法上

@Autowired

@Resource

@AliasFor

注解的属性别名，如果注解放在 value 和 path 上，相当于 value 和 path 相同

2021年我该如何进行学习？这是我对自己技术的一个迷茫点。
首先，我给自己列个几个不同的方向

  1. 继续钻研算法——刷爆leetcode，参加各大OJ比赛，研究算法，数学
  2. 学习框架使用——其实自己对于框架的使用一直都不是很熟练，因为没有特别花心思在上面研究，只会一些基础的理论知识，实际上对于特别原理性的东西还是一知半解，包括spring的注解，aop的使用场景等。使用spring到底可以给我自己带来什么样的好处？没有spring的话会很麻烦嘛？思来想去，感觉ssm的作用就那么一点，虽然涵盖了工程上的大部分适用场景，包括生成对象，路由转发，查库啥的，然而我不是特别理解。这部分就是多做几个项目去熟悉。
  3. 学习系统设计、学习中间件，在架构的角度看系统，看系统的设计合理性，如何设计分布式系统等等。
  4. 学习类设计，让代码更加厉害。
  5. 学习底层知识，操作系统，计算机网络等等这些内功心法
  6. 学习并发编程，网络编程，这属于搞java核心编程了，也是让自己代码变的更厉害，让自己可以看懂更高级的代码

| 算法 | 3 | 5 | 2 | 1 |
| 核心编程 | 2 | 3 | 4 | 4 |
| 架构设计 | 4 | 2 | 3 | 2 |
| 底层知识 | 5 | 1 | 5 | 3 |
| 框架使用 | 1 | 4 | 1 | 5 |

5475. 统计好三元组

题目描述：

给你一个整数数组 arr ，以及 a、b 、c 三个整数。请你统计其中好三元组的数量。
如果三元组 (arr[i], arr[j], arr[k]) 满足下列全部条件，则认为它是一个 好三元组 。
0 <= i < j < k < arr.length
|arr[i] - arr[j]| <= a
|arr[j] - arr[k]| <= b
|arr[i] - arr[k]| <= c
其中 |x| 表示 x 的绝对值。
返回 好三元组的数量
